ul li : 순서가 없는 리스트

  • ul 태그는 순서가 없는 리스트 작성시 부모로써 가장 먼저 작성
    li태그는 그의 자녀 요소로 작성!
  • li태그는 다음에 오는 요소가 li요소로 형제가 정해져있음!
  • 헤드라인태그는 h2~h6까지 사용이 가능
    div태그를 사용할수 있음!
  • ul태그는 순서가 없기 때문에 맨앞에 점이 붙는다.
    점이 붙거나 숫자가 붙거나 하는 것들은 블릿기호라고 말한다.

dl (definition list) : 직접정의 내린 리스트

용어정리

dl 리스트 특징 ( 인라인과 블록요소가 섞여있음)

dt가 인라인 요소( definition term ) : 직접 정의 내린 제목
dd는 블록 요소 ( definition description) : 직접 정의 내린 설명글

dl 리스트 사용하기

반드시 dt, dd를 전부 사용해야 함

li태그들은 블록요소, div태그도 담을 수 있음
dd태그도 마찬가지로 li태그와 사용방법이 같음

반드시 dt, dd를 사용해야 하기 때문에
제목이 있는 페이지를 만들때 사용한다.
주로 상세페이지 제작시 사용
ul li / ol li 의 경우에는 각각의 기능이 정확하게
있기 때문에 점이나 숫자가 있지만..
dl의 경우에는 블릿기호가 없음( 사용자가 직접 정의! )
